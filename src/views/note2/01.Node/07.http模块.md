# http 模块

---

## 一、概述

`http` 模块建立在 `net` 模块之上，无须手动管理 socket，无须手动组装 HTTP 消息格式。

`http` 模块既可以作为**客户端**发起请求，也可以作为**服务器**接收请求。


| 角色  | 核心 API                | 请求对象              | 响应对象              |
| --- | --------------------- | ----------------- | ----------------- |
| 服务器 | `http.createServer()` | `IncomingMessage` | `ServerResponse`  |
| 客户端 | `http.request()`      | `ClientRequest`   | `IncomingMessage` |


> 请求和响应体都是以**流（Stream）**的形式传输的，需要通过 `data` / `end` 事件逐块读取。

---

## 二、服务器端

### 2.1 创建服务器

```js
const http = require("http");

const server = http.createServer((req, res) => {
  // req: IncomingMessage — 请求对象
  // res: ServerResponse   — 响应对象
});

server.listen(9527);

server.on("listening", () => {
  console.log("server listen 9527");
});
```

### 2.2请求信息和响应信息

`req` 是 `IncomingMessage` 对象，常用属性：


| 属性 / 方法       | 说明                          |
| ------------- | --------------------------- |
| `req.url`     | 请求路径（含 query），如 `/api?id=1` |
| `req.method`  | 请求方法，如 `GET`、`POST`         |
| `req.headers` | 请求头对象                       |


`res` 是 `ServerResponse` 对象，常用方法：


| 方法 / 属性                      | 说明                  |
| ---------------------------- | ------------------- |
| `res.setHeader(name, value)` | 设置响应头               |
| `res.statusCode`             | 设置状态码，如 `200`、`404` |
| `res.write(data)`            | 向响应体写入内容（可多次调用）     |
| `res.end(data?)`             | 结束响应，可选传入最后一段内容     |


### 2.3 完整代码

```js
const http = require("http");
const url = require("url");

// 处理请求参数
const handleRequest = (req) => {
    console.log("有请求来了");
    const urlobj = url.parse(req.url);
    console.log("请求路径", urlobj);
    console.log("请求方法", req.method);
    console.log("请求头", req.headers);

    let body = "";
    req.on("data", chunk => {
        body += chunk.toString("utf-8");
    });

    req.on("end", () => {
        console.log("请求体", body);
    });
}

/* 
创建服务器
请求是 IncomingMessage对象
响应是 ServerResponse对象
*/
const server = http.createServer((req, res) => {
    handleRequest(req);//处理请求对象
    res.setHeader("a", "hello");//设置响应头
    res.statusCode = 404;//设置响应状态码
    res.write("你好，客户端！");//响应体写入内容
    res.end();//结束响应
})

// 监听服务器的listening事件
server.on("listening", () => {
    console.log("server listen 9527");
});

//监听9527端口
server.listen(9527);
```

## 三、客户端

### 3.1 发起请求

使用 `http.request()` 发起 HTTP 请求：

```js
const http = require("http");

const request = http.request(
  "http://www.baidu.com",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
  },
  (resp) => {
    // resp: IncomingMessage — 响应对象
    console.log(`STATUS: ${resp.statusCode}`);
    console.log(`HEADERS: ${JSON.stringify(resp.headers)}`);
  }
);
```

### 3.2 完整代码

```js
const http = require("http");

/* 
客户端-请求其它接口
请求是 ClientRequest对象
响应式 IncomingMessage对象
*/
const request = http.request("http://www.baidu.com",
    {
        method: "POST",
        headers: { 'Content-Type': 'application/json' }
    },
    (resp) => {
        console.log(`STATUS: ${resp.statusCode}`);
        console.log(`HEADERS: ${JSON.stringify(resp.headers)}`);

        // resp是以流的形式返回的
        let result = "";

        resp.on("data", chunk => {
            result += chunk.toString("utf-8");
        })

        resp.on("end", chunk => {
            console.log(JSON.parse(result));
        });
    })

// 监听错误事件
request.on('error', (e) => {
    console.error(`错误信息==>: ${e.message}`);
});


// 往请求体中添加数据且结束请求
request.write(JSON.stringify({ msg: "hello goole" }));
request.end();
```

## 四、实践 — 静态资源服务器

对应练习文件：`case.js`

根据请求路径，在服务器内部查找对应文件，找到则返回文件内容，找不到则返回 404。

### 4.1 整体思路

```
客户端请求 /index.css
       ↓
解析 url，得到 pathname
       ↓
拼接本地路径 static/index.css
       ↓
判断文件是否存在 → 不存在 → 404
       ↓
是目录 → 尝试读取目录下的 index.html
       ↓
读取文件内容 → 200 响应
```

### 4.2 目录结构

```
homework/
├── case.js          # 静态资源服务器
├── index.js         # 基础 HTTP 服务器
├── client.js        # HTTP 客户端
└── static/          # 静态资源目录
    ├── index.html
    ├── index.css
    └── hello.js
```

 启动 `case.js` 后，访问 `http://localhost:9527/index.html` 即可加载页面及其引用的 CSS、JS 资源

### 完整代码

```js
/* 
实践-静态资源服务器
* 根据请求路径，寻找服务器内部是否存在资源，响应给客户端
*/

const http = require('http');
const url = require('url');
const path = require('path');
const fs = require('fs');

// 获取文件信息(判断文件是否存在)
const getStat = async (filename) => {
    try {
        return await fs.promises.stat(filename);
    } catch {
        return null;
    }
}

// 获取文件内容
const getFileContent = async (req) => {
    const urlObj = url.parse(req.url);

    let fileName = path.resolve(__dirname, 'static', urlObj.pathname.substring(1))
    console.log("fileName", fileName);
    const stat = await getStat(fileName);//获取文件信息

    // 文件不存在
    if (!stat) {
        return null;
    }

    // 文件是一个目录
    if (stat.isDirectory()) {
        fileName = path.resolve(__dirname, "static", urlObj.pathname.substring(1), 'index.html');
        const stat = await getStat(fileName);
        if (!stat) { return null }
        return await fs.promises.readFile(fileName)
    }

    // 直接读取文件
    return await fs.promises.readFile(fileName)
}

// 处理响应
const handler = async (req, res) => {
    const info = await getFileContent(req);

    if (!info) {
        res.statusCode = 404;
        res.end("资源不存在", "utf-8");
        return;
    }

    res.statusCode = 200;
    res.write(info);
    res.end();
}


// 创建服务器
const server = http.createServer(handler)

server.on("listening", () => {
    console.log("server listen 9527");
})

server.listen(9527);
```
