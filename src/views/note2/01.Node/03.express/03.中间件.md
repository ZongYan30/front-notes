# 中间件
## 理解中间件
<img src="https://zongyan30.github.io/front-assets/Node/中间件示意图.jpg"/>

```js
const express = require("express");
const app = express();

/* 中间件 */
app.get("/news", (req, res, next) => {
  console.log("handler1");
  next();
});

app.get("/news", (req, res, next) => {
  console.log("handler2");
  res.end();//这里虽然end了，但是后续的中间件依旧会执行
  next();
});

app.get("/news", (req, res, next) => {
  console.log("handler3");
  res.send("handler3");//这里就会失效，因为上边已经end了。

  // throw new Error("abc");//相当于 next(new Error("abc"))
  next(new Error("abc"));//抛出错误但放行
});

/* 监听端口 */
app.listen(9527, () => {
  console.log(`server listen on 9527`);
});

```

中间件规则：
* 当匹配到了请求后,交给第一个处理函数处理,函数中需要手动的交给后续中间件处理。
* 如果后续已经没有了中间件,express发现如果响应没有结束，express会响应404
* 如果中间件发生了错误,不会停止服务器,相当于调用了 next(错误对象)。寻找后续的错误处理中间件。
* 如果没有，则响应500

## use中间件
```js
/* 对所有请求都应用中间件 */
app.use();

/* 
能匹配  /news  /news/abc   /news/123   /news/ab/adfs
不能匹配  /n   /a   /   /newsabc
*/
app.use("/news",require("./middleware"));
```

## 自定义中间件
```js
// 自定义了一个错误处理的中间件
module.exports = (err, req, res, next) => {
  if (err) {
    const errObj = {
      code: 500,
      msg: err instanceof Error ? err.message : err,
    };
    //发生了错误
    res.status(500).send(errObj);
  } else {
    next();
  }
};
```

## 工程目录结构
```
├─ models
│  ├─ db.js    数据库连接
│  ├─ sync.js  数据库同步
│  ├─ init.js  数据库初始化
│  └─ user.js  用户模型
├─ routes
│  ├─ xxx.js  xx路由
│  └─ init.js  路由初始化
├─ services
│  ├─ userServices.js  用户服务层
│  └─ init.js  服务初始化
├─ routes
│  ├─ init.js  路由初始化
│  └─ loginRoutes.js  登录路由
├─ logs
│  └─ sql.log  登录日志
├─  index.js
└─  init.js
```
