# 数据查询

## 1. `findOne`

用途：查询满足条件的第一条记录，常用于登录校验、按唯一字段查找等场景。

代码示例：

```js
const result = await Admin.findOne({
	where: { loginId, loginPwd }
});
```

说明：

- `where` 用于设置查询条件。
- 上面会生成“账号和密码同时匹配”的查询。
- 如果查到了数据，返回的是一个 Sequelize 实例对象。
- 如果没查到，返回 `null`。
- 想要得到普通对象，可以使用 `result.toJSON()`。

在示例中的用途：

```js
exports.login = async (loginId, loginPwd) => {
	const result = await Admin.findOne({
		where: { loginId, loginPwd }
	});

	if (result && result.loginId === loginId && result.loginPwd === loginPwd) {
		return result.toJSON();
	}

	return null;
}
```

## 2. `findByPk`

用途：根据主键查询一条记录，适合通过 `id` 查详情。

代码示例：

```js
const ins = await Admin.findByPk(id);
```

说明：

- `Pk` 是 `Primary Key`，表示主键。
- 一般会根据表的主键字段查询，常见就是 `id`。
- 查询到后返回 Sequelize 实例对象。
- 查询不到返回 `null`。
- 若要返回普通对象，可调用 `toJSON()`。

在示例中的用途：

```js
exports.getAdminById = async (id) => {
	const ins = await Admin.findByPk(id);
	if (!ins) { return null };
	return ins.toJSON();
}
```

## 3. `findAll`

用途：查询所有满足条件的记录；如果不传条件，就是查询整张表的数据。

代码示例：

```js
const users = await Admin.findAll();
```

说明：

- 返回值是数组。
- 即使查不到数据，返回的也是空数组 `[]`，而不是 `null`。
- 数组中的每一项都是 Sequelize 实例对象。
- 如果要转成普通对象数组，通常配合 `map(item => item.toJSON())`。

在示例中的用途：

```js
exports.getAllAdmins = async () => {
	const users = await Admin.findAll();
	if (!users) { return null };
	return users.map(user => user.toJSON());
}
```

对应学生表查询：

```js
exports.getAllStudents = async () => {
	const students = await Student.findAll();
	if (!students) { return null };
	return students.map(item => item.toJSON());
}
```

## 4. `findAll` 配合分页

`studentService.js` 中还演示了分页查询的写法：

```js
const result = await Student.findAll({
	attributes: ["id", "name", "sex"],
	where,
	limit: +size,
	offset: (page - 1) * size,
});
```

说明：

- `attributes`：指定要查询的字段，避免把所有列都查出来。
- `where`：动态条件对象，用于筛选数据。
- `limit`：限制本次查询返回的条数。
- `offset`：跳过前面多少条数据，通常和分页一起使用。
- 第 `page` 页每页 `size` 条时，偏移量公式是：`(page - 1) * size`。

## 5. `where` 条件查询

代码中通过对象动态拼接查询条件：

```js
const where = {};

if (sex != -1) {
	where.sex = !!sex;
}

if (name) {
	where.name = {
		[Op.like]: `%${name}%`
	}
}
```

说明：

- `where` 本质上是一个普通对象，用来描述 SQL 的查询条件。
- `where.sex = !!sex`：将传入值转成布尔值后再查询。
- `[Op.like]`：Sequelize 提供的操作符，表示模糊查询。
- `` `%${name}%` ``：表示“名字中包含某个关键字”。

等价理解：

- `sex` 条件存在时：按性别筛选。
- `name` 条件存在时：执行 SQL 中类似 `name like '%关键字%'` 的效果。

## 6. 统计总数 `count`

分页查询不仅要查当前页数据，还要知道总记录数：

```js
return {
	datas: result.map(item => item.toJSON()),
	total: await Student.count()
}
```

说明：

- `Student.count()` 用于统计表中的记录总数。
- 常用于前端分页，前端需要根据总数计算总页数。
- 当前代码里 `total` 统计的是整张表总数，没有带上 `where` 条件。
- 如果希望“总数”和筛选条件一致，通常应该写成：

```js
const total = await Student.count({ where });
```

---

## 7. 常见返回值总结

- `findOne`：返回单个实例，查不到返回 `null`
- `findByPk`：返回单个实例，查不到返回 `null`
- `findAll`：返回实例数组，查不到返回空数组 `[]`
- `count`：返回数字，表示记录总数

---

## 8. 本次代码中的查询 API 总结

| API | 用途 |
| --- | --- |
| `findOne` | 查询满足条件的第一条记录，常用于登录查询 |
| `findByPk` | 根据主键查询单条记录 |
| `findAll` | 查询多条记录或整张表数据 |
| `findAll + where + limit + offset` | 条件筛选并分页查询 |
| `count` | 统计记录总数 |
