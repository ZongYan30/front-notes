# 数据库简介

## 数据库能做什么？

- **持久存储数据**：数据保存在硬盘文件中，断电不丢失
- **备份与恢复**：支持数据的备份、还原与迁移
- **快速存取**：通过索引、缓存等机制高效读写
- **权限控制**：限制不同用户对数据的访问与操作范围

## 关系数据库 vs 非关系数据库

| 维度       | 关系数据库（RDBMS）                          | 非关系数据库（NoSQL）         |
| -------- | ------------------------------------- | --------------------- |
| **特点**   | 以表（Table）及表之间的关联构成数据结构                | 结构简单，常见形式有文档型、键值对等    |
| **优点**   | 能表达复杂的数据关系；支持强大的查询语言（如 SQL）；可精确查找目标数据 | 格式灵活；海量数据读写效率高        |
| **缺点**   | 读写性能相对较差，尤其在海量数据场景下；表结构相对固定           | 难以表示复杂的数据结构；复杂查询效率较低  |
| **适用场景** | 存储结构复杂、关联关系明确的数据                      | 存储结构简单、读写量大、扩展性要求高的数据 |
| **代表产品** | Oracle、MySQL、SQL Server               | MongoDB、Redis、Membase |

## 术语

| 缩写   | 英文全称                       | 中文含义                  |
| ---- | -------------------------- | --------------------- |

## 使用

### 进入 MySQL 命令行

```sh
mysql -uroot -p
```

### 查看字符编码

```sh
show variables like 'character\_set\_%';
```

修改 `my.ini` 中的默认字符编码（路径示例：`C:\ProgramData\MySQL\MySQL Server 8.0`），修改后重启服务：

```sh
net stop mysql80
net start mysql80
```

### 查看数据库列表

```sh
show databases;
```
