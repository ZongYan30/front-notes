## 模块的查找
**1.绝对路径**
```js
require('/D:/file.js'); // 绝对路径
```

**2.相对路径**
相对于当前模块，转换为绝对路径，加载模块
```js
require('./file.js'); // 相对路径
```

**3.相对路径2**
* 检查是否是内置模块，如：fs、path等
* 检查当前目录中的node_modules
* 检查上级目录中的node_modules
* 转换为绝对路径
* 加载模块
```js
require('fs'); // 内置模块
```

**4.文件和后缀名查找**
* 如果不提供后缀名，自动补全 (js、json、node、mjs);
* 如果仅提供目录，不提供文件名，则自动寻找该目录中的index.js
* package.json中的main字段
  表示包的默认入口
  导入或执行包时若仅提供目录，则使用main补全入口
  默认值为index.js

```js
require('./file');//默认补全file.js
require('./file/');//默认补全index.js
```

假设a包的package.json文件内容如下：
```json
{
  "main": "dist/index.js"
}
```
则require('a')时会从main字段中找入口

## module对象
记录当前模块的信息，父模块是谁，子模块有哪些，模块的导出对象等
如：exports、filename、loaded、parent、children、paths

```js
console.log(module);
```

## require函数
require函数用于加载模块，返回模块的导出对象
```js
console.log(require);
```
> require函数下有一个resolve方法，用于解析模块的绝对路径，偶尔会用到

## require函数的大致工作原理
伪代码阐述流程
```js
function require(modulePath) {
    //1. 将modulePath转换为绝对路径：D:\repository\NodeJS\源码\myModule.js

    //2. 判断是否该模块已有缓存
    if (require.cache["D:\\repository\\NodeJS\\源码\\myModule.js"]) {
        return require.cache["D:\\repository\\NodeJS\\源码\\myModule.js"].result;
    }

    //3. 读取文件内容

    //4. 包裹到一个函数中
    function __temp(module, exports, require, __dirname, __filename) {
        console.log("当前模块路径：", __dirname);
        console.log("当前模块文件：", __filename);
        exports.c = 3;
        module.exports = {
            a: 1,
            b: 2
        };
        this.m = 5;
    }

    //6. 创建module对象
    module.exports = {};
    const exports = module.exports;//在最开始module.exports和exports指向同一个对象

    __temp.call(module.exports, module, exports, require, module.path, module.filename)
    return module.exports;
}

// require.cache = {};

const result = require("./myModule");
console.log(result);
```

面试题
```js [a.js]
console.log("当前模块路径：", __dirname);
console.log("当前模块文件：", __filename);

this.m = 5;//this最开始指向是module.exports对象
exports.c = 3;//exports最开始指向是module.exports对象

module.exports = {//这里重新给module.exports赋值了，导致了module.exports和exports指向不同的对象
  a: 1,
  b: 2
};
```

问：引入模块输出什么?
```js
const result = require('./a.js');
console.log(result);//{ a: 1, b: 2}
```

> 可以理解为this就是exports对象，但绝不是module.exports对象

**在日常开发中要只用一种风格的导出**
