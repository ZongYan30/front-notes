# 事件循环

核心就一张图：
<img src="https://zongyan30.github.io/front-assets/Node/EventLoop.png"/>

事件循环中存在几个队列：

1. timers: 存放计时器的回调函数
2. poll：轮询队列
   除了timers、checks,绝大部分回调都会放入该队列,比如：文件的读取、监听用户请求
   运作方式:
   - 如果poll中有回调，依次执行回调，直到清空队列
   - 如果poll中没有回调,等待其他队列中出现回调，结束该阶段，进入下一阶段
   - 如果其他队列也没有回调，持续等待，直到出现回调为止
3. check：检查阶段,使用setImmediate的回调会直接进入这个队列

还有两个优先级最高的：

1. nextTick：微任务队列,用于执行process.nextTick的回调函数
2. promise：微任务队列,用于执行Promise的回调函数

> 事件循环中，每次打算执行一个回调之前，必须要先清空nextTick和promise队列

## 代码分析

:::code-group

```js [1.js]
let i = 0;
console.time();
function test() {
  i++;
  if (i < 1000) {
    setImmediate(test); //可换setTimeout
  } else {
    console.timeEnd();
  }
}

test();
/* 输出:
default: 5.007ms  //换成setTimeout会增加很长的执行时间
*/
```

```js [2.js]
setTimeout(() => {
  console.log("setTimeout");
}, 0);

setImmediate(() => {
  console.log("setImmediate");
});
/* 
输出:
setImmediate
setTimeout
*/
```

```js [3.js]
const start = Date.now();
setTimeout(function f1() {
  console.log("setTimeout", Date.now() - start);
}, 200);

const fs = require("fs");

fs.readFile("./index.js", "utf-8", function f2() {
  console.log("readFile");
  const start = Date.now();
  while (Date.now() - start < 300) {}
});
/* 
输出：
readFile
setTimeout 307
*/
```

```js [面试题1.js]
const fs = require("fs");
fs.readFile("./index.js", () => {
  setTimeout(() => console.log(1), 0);
  setImmediate(() => console.log(2));
});
//2 1
```
