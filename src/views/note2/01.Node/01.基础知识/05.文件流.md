# Node.js 文件流 —— 可读流（createReadStream）

---

## 一、流的概念

流是指数据的流动，数据从一个地方缓缓的流动到另一个地方。

流是有方向的：

- 可读流: Readable 数据从源头流向内存
- 可写流: Writable 数据从内存流向源头
- 双工流：Duplex 数据即可从源头流向内存 又可从内存流向源头

为什么需要流？

1. 其他介质和内存的数据规模不一致 （内存空间小，但速度快）
2. 其他介质和内存的数据处理能力不一致(内存读取块，其他读取慢)

什么是文件流？

内存数据和磁盘文件数据之间的流动

## 二、可读流&#x20;

```js
fs.createReadStream();
```

### 2.1 基本使用

```js
const fs = require("fs");
const path = require("path");

const filename = path.resolve(__dirname, "./a.txt");

const rs = fs.createReadStream(filename, {
  encoding: "utf-8", // 编码
  highWaterMark: 1, // 每次读取多少数据，默认 64 * 1024
  autoClose: true, // 读取完毕后自动关闭文件
});
```

### 2.2 配置项速查表

| 配置项          | 类型      | 默认值              | 说明                                        |
| --------------- | --------- | ------------------- | ------------------------------------------- |
| `encoding`      | `string`  | `null`              | 指定编码，如 `"utf-8"`。不指定则返回 Buffer |
| `highWaterMark` | `number`  | `64 * 1024`（64KB） | 每次读取的数据量（字节数）                  |
| `start`         | `number`  | `0`                 | 从文件的第几个字节开始读取                  |
| `end`           | `number`  | 文件末尾            | 读取到文件的第几个字节结束                  |
| `autoClose`     | `boolean` | `true`              | 读取完毕后是否自动关闭文件描述符            |
| `fd`            | `number`  | —                   | 手动指定文件描述符，与 `path` 互斥          |

---

### 2.3 事件速查表

| 事件名   | 触发时机                        | 回调参数           |
| -------- | ------------------------------- | ------------------ |
| `open`   | 文件被打开时                    | `fd`（文件描述符） |
| `data`   | 每次读取到数据块时              | `chunk`（数据块）  |
| `pause`  | 流被暂停时（调用 `pause()`）    | 无                 |
| `resume` | 流恢复读取时（调用 `resume()`） | 无                 |
| `end`    | 所有数据读取完成时              | 无                 |
| `close`  | 文件被关闭时                    | 无                 |
| `error`  | 读取过程中出错时                | `err`（错误对象）  |

---

### 2.4 事件速查

| 方法                   | 说明                                      |
| ---------------------- | ----------------------------------------- |
| `rs.pause()`           | 暂停读取，停止触发 `data` 事件            |
| `rs.resume()`          | 恢复读取，重新开始触发 `data` 事件        |
| `rs.destroy([error])`  | 销毁流，可选传入错误对象触发 `error` 事件 |
| `rs.pipe(destination)` | 将可读流管道连接到可写流                  |

---

### 2.5 练习

```js
const fs = require("fs");
const path = require("path");

const filename = path.resolve(__dirname, "./a.txt");

// 创建读取流
const rs = fs.createReadStream(filename, {
  encoding: "utf-8", // 编码
  highWaterMark: 1, // 每次读取多少数据 默认64 * 1024
  // start: 0,            // 从0开始读取
  // end: 10,             // 读取到10结束
  autoClose: true, // 读完后会自动关闭文件
});

// 监听-打开文件
rs.on("open", () => {
  console.log("文件被打开了");
});

// 监听-出错
rs.on("error", () => {
  console.log("出错了！！");
});

// 监听-关闭文件 (读取完之后会自动关闭文件)
rs.on("close", () => {
  console.log("文件关闭了");
});

// 监听-数据流
rs.on("data", (chunk) => {
  console.log("读到了一部分数据：", chunk);
  rs.pause(); // 暂停读取
});

// 监听-文件暂停
rs.on("pause", () => {
  console.log("暂停了");
  setTimeout(() => {
    rs.resume();
  }, 1000);
});

// 监听-文件恢复读取
rs.on("resume", () => {
  console.log("恢复了");
});

// 监听-文件读取完毕
rs.on("end", () => {
  console.log("全部数据读取完毕");
});
```

## 三、可写流（createWriteStream）

```js
fs.createWriteStream(path, options);
```

### 3.1 基本使用

```js
const fs = require("fs");
const path = require("path");

const filename = path.resolve(__dirname, "./output.txt");

const ws = fs.createWriteStream(filename, {
  encoding: "utf-8",
  highWaterMark: 16 * 1024, // 默认 16KB
});

ws.write("hello");
ws.write(" world");
ws.end("!"); // 标记写入结束
```

### 3.2 配置项速查表

| 配置项          | 类型      | 默认值              | 说明                                            |
| --------------- | --------- | ------------------- | ----------------------------------------------- |
| `encoding`      | `string`  | `"utf-8"`           | 写入时的编码格式                                |
| `highWaterMark` | `number`  | `16 * 1024`（16KB） | 内部缓冲区大小（字节数）                        |
| `flags`         | `string`  | `"w"`               | 文件打开模式，如 `"w"` 覆盖写入、`"a"` 追加写入 |
| `autoClose`     | `boolean` | `true`              | 写入完毕后是否自动关闭文件描述符                |
| `start`         | `number`  | —                   | 从文件的第几个字节开始写入                      |

---

### 3.3 可写流 —— 事件速查表

| 事件名   | 触发时机                          | 回调参数           |
| -------- | --------------------------------- | ------------------ |
| `open`   | 文件被打开时                      | `fd`（文件描述符） |
| `drain`  | 缓冲区排空，可以继续写入时        | 无                 |
| `finish` | 调用 `end()` 且所有数据写入完成后 | 无                 |
| `close`  | 文件被关闭时                      | 无                 |
| `error`  | 写入过程中出错时                  | `err`（错误对象）  |

---

### 3.4 可写流 —— 方法速查表

| 方法                  | 返回值    | 说明                                                         |
| --------------------- | --------- | ------------------------------------------------------------ |
| `ws.write(chunk)`     | `boolean` | 写入数据。返回 `false` 表示缓冲区已满，需要等待 `drain` 事件 |
| `ws.end([chunk])`     | —         | 标记写入结束，可选传入最后一块数据                           |
| `ws.destroy([error])` | —         | 销毁流，可选传入错误对象触发 `error` 事件                    |

> **关于 write() 返回值（背压 Backpressure）：**
>
> - 返回 `true`：数据已直接写入目标，可以继续写入
> - 返回 `false`：内部缓冲区已满，应暂停写入，等待 `drain` 事件后再继续

## 四、文件复制实战

### 4.1 方式一：直接读写

```js
const fs = require("fs");
const path = require("path");

async function method1() {
  const from = path.resolve(__dirname, "./bigFile.txt");
  const to = path.resolve(__dirname, "./bigFile2.txt");

  console.time("方式1");
  const content = await fs.promises.readFile(from);
  await fs.promises.writeFile(to, content);
  console.timeEnd("方式1");
  console.log("复制完成");
}
```

> **缺点：** 需要将整个文件一次性读入内存，大文件会占用大量内存，甚至导致内存溢出。

### 4.2 方式二：流式背压控制

<img src="https://zongyan30.github.io/front-assets/Node/beiya2.png"/>

```js
const fs = require("fs");
const path = require("path");

async function method2() {
  const from = path.resolve(__dirname, "./bigFile.txt");
  const to = path.resolve(__dirname, "./bigFile2.txt");

  console.time("方式2");

  // 创建流，以 Buffer 的方式
  const rs = fs.createReadStream(from);
  const ws = fs.createWriteStream(to);

  rs.on("data", (chunk) => {
    const flag = ws.write(chunk);
    // 产生背压，停止读取
    if (!flag) {
      rs.pause();
    }
  });

  ws.on("drain", () => {
    // 管道释放 —— 继续写入
    rs.resume();
  });

  // 读完了
  rs.on("close", () => {
    ws.end(); // 结束写入流
    console.timeEnd("方式2");
    console.log("复制完成");
  });
}
```

> **背压（Backpressure）流程图解：**
>
> **背压核心思路：**
>
> 1. 可读流读取数据通过 `data` 事件写出
> 2. 每次调用 `ws.write(chunk)` 写入数据，并检查返回值
> 3. 若返回 `false`，说明可写流内部缓冲区已满，此时调用 `rs.pause()` 暂停读取
> 4. 当可写流缓冲区排空后，会触发 `drain` 事件
> 5. 在 `drain` 事件中调用 `rs.resume()` 恢复读取，继续写入

### 4.3 方式三：pipe 管道

```js
const fs = require("fs");
const path = require("path");

async function method3() {
  const from = path.resolve(__dirname, "./bigFile.txt");
  const to = path.resolve(__dirname, "./bigFile2.txt");

  console.time("方式3");

  const rs = fs.createReadStream(from);
  const ws = fs.createWriteStream(to);

  rs.on("end", () => {
    console.timeEnd("方式3");
    console.log("复制完成");
  });

  rs.pipe(ws);
}
```

> `pipe()` 自动处理了背压控制，相当于方式二的简化版，推荐使用。
